# Showcase — a complete end-to-end run

This folder is the proof. It shows `hire-me-pls` applied to one fictional candidate, one real-looking role, from career file to finished application.

The candidate is **Jordan Park**, a senior product manager moving from B2C consumer apps to B2B SaaS. The target role is **Senior Product Manager, Workflow Automation** at the fictional **Northwind Operations**.

Everything here is invented. Jordan Park is not a real person; Northwind Operations, Beta Inc, and Alpha Apps are not real companies. The point is to show the *shape* of a good run, not to leak anyone's real data.

## What this folder contains

This is a complete, frozen run of the framework — every artifact, generated by the actual skills and scripts.

| File | What it demonstrates |
| --- | --- |
| `candidate-profile.md` | The free-form career file the framework reads as its source of truth |
| `config.yaml`, `branches.yaml`, `regional-headers.yaml` | A real, filled-in configuration — what `job-search-setup` writes |
| `voice/cover-letter-sample.txt` | The voice reference the cover-letter skill anchors on |
| `Diagnosis - Northwind Operations - Senior PM.md` | The five-section diagnosis that drives everything downstream |
| `CV - Northwind Operations - Senior PM.docx` | The rendered, ATS-optimized CV — produced via the OPUS template and the docxtpl pipeline, and passed the post-render audit |
| `Cover Letter - Northwind Operations - Senior PM.docx` | The voice-anchored cover letter, 250 words, no em dashes: a tight specific opener, the strongest proof as plain description, the B2C-to-B2B objection named and resolved, and a warm close |
| `Northwind Operations - Senior PM - Interview Prep.docx` | Role snapshot, four-theme STAR+R story map, three hard questions, three questions to ask back |
| `LinkedIn Messages.txt` | The sub-80-word recruiter nudge |

## The audit result

The CV was checked by the programmatic post-render audit before being placed here. All checks passed:

- **Check 2 (keywords):** 9 of 10 JD keywords appear verbatim ("mid-market" was the one term that did not earn a natural place — well above the two-keyword floor).
- **Check 4 (ampersands):** clean — no stripped `&`.
- **Check 5 (bold runs):** bold runs are present in the experience section, as the diagnosis specified. The editorial bolding is deliberately disciplined — exactly five bolded phrases across the whole CV ("7-day activation from 32% to 38%", "TechCrunch", "three weeks to six days", "22%", "second-most-used feature"), every one a quantified outcome or a credential proper noun, none a JD keyword.

## How to read the showcase

1. Start with `candidate-profile.md` — what Jordan handed the framework. Notice it is free-form prose, not a filled-in schema. That is the intended input format.
2. Read `Diagnosis - Northwind Operations - Senior PM.md` — the diagnosis the framework wrote from the JD. It does not summarize the JD; it reads it.
3. Open `CV - Northwind Operations - Senior PM.docx` next to the diagnosis. Every bullet traces back to a line in the diagnosis. `role-diagnosis/examples/saas-pm-role.md` walks through that mapping exp
